Why Node Security Matters

Relay nodes are part of NOIZ’s managed infrastructure mesh. If a node is compromised, it can:

  • Drop or corrupt stream packets
  • Leak routing metadata or handshake details
  • Degrade platform reliability
  • Trigger auto-blacklisting or loss of ⚡︎dB rewards

While nodes never access personal viewer data or raw stream content, good security practices protect:

  • Viewer privacy
  • Consistent performance
  • Pathweaver eligibility for rewards and milestones

Core Security Principles

  • Host relays in stable, controlled environments
  • Use only NOIZ-signed binaries and official updates
  • Restrict network exposure via firewall and allowed ports
  • Keep relay software fully updated
  • Never alter routing code or attempt to force traffic prioritization

System-Level Hardening

Area Best Practice
OS & Patching Use LTS OS versions (e.g., Ubuntu LTS) and update regularly
Firewall Allow only required ports (3 default, defined per binary)
User Privileges Run under a non-root user account
Logging Keep only basic relay logs — never store payload or headers
SSH Access Disable password login, use SSH keys only
Monitoring Use uptime monitors to verify health, not inspect content

NOIZ does not manage your server environment — operators are responsible for system-level maintenance.

Token & Config Security

  • Treat your Node Token like a password — do not share it
  • Never post configs or tokens publicly (e.g., GitHub repos)
  • Restrict config file access to the relay user only
  • Rotate tokens (--rotate-token) immediately if compromised

⚠️ Exposing or embedding your token in public scripts is grounds for instant relay removal.

Preventing Abuse

Relays must not:

  • Route traffic for unrelated video services (e.g., YouTube, Kick)
  • Replay or locally cache stream data
  • Alter source headers or fake latency/performance metrics
  • Inspect encrypted payloads — relays forward content only

Attempts to bypass these rules may result in blacklisting and forfeiture of ⚡︎dB rewards.

Monitoring Safely

Allowed:

  • Use NOIZ’s web dashboard for performance metrics
  • Run private Grafana/Prometheus monitors
  • Enable private alerts for downtime or crashes

Not allowed:

  • Public monitoring dashboards
  • Embedding nodes in public CDNs or free hosting layers
  • Auto-scaling with unvetted IP ranges or unknown instances

Incident Response

If NOIZ flags your node:

  • You’ll see a dashboard alert with issue details
  • Severe issues may trigger auto-disable or token revocation
  • You may:
  • Patch your configuration
  • Rotate credentials
  • Reapply after the required cool-down period

FAQ

  • Can I run behind Cloudflare or VPN? → No — nodes must be directly routable for live stream paths.
  • Can I host multiple relays on one machine? → Not recommended — can cause port and stability conflicts.
  • Can I run on public or shared computers? → No — violates operational security standards.
  • Can I modify the UI or handshake logs? → No — relays are a secure, closed routing layer.
