1. Why Security and Privacy Matter

Plugins are powerful tools that can:

  • React to live stream events
  • Interact with viewers and chat
  • Trigger overlays, visuals, and sounds
  • Manage ⚡︎dB-based triggers

But they also introduce risks if mishandled, including:

  • Stream disruption or instability
  • Viewer annoyance, harm, or exploitation
  • Unauthorized data collection
  • Platform policy or legal violations

These guidelines exist to protect streamers, viewers, and developers while maintaining a safe, trusted plugin ecosystem.

2. Security Requirements

All plugins must:

  • Run in a sandboxed environment (no root-level or OS access)
  • Use only the NOIZ-provided SDK APIs
  • Avoid accessing browser internals (DOM, cookies, storage, etc.)
  • Fail gracefully if a hook or API is unavailable
  • Avoid creating persistent outbound connections unless pre-approved

Plugins must not:

  • Inject JavaScript into other plugin layers
  • Trigger full-page overlays or site redirects
  • Open pop-ups, elevate iframes, or hijack browser permissions
  • Access stream content, viewer IPs, or external domains without explicit approval

3. Privacy Rules

Plugins must never:

  • Record user input (keyboard, mouse, microphone)
  • Log viewer behavior (clicks, dwell time, scrolling patterns)
  • Attempt to infer identity from geography, unlock patterns, or behavior
  • Store unique identifiers in cookies or local storage
  • Use fingerprinting, tracking pixels, or hidden analytics scripts

All interaction must be:

  • Voluntary
  • Session-scoped only (no persistent tracking)

4. ⚡︎dB Trigger Safeguards

If your plugin uses ⚡︎dB for triggers or effects:

  • Triggers must be viewer-initiated (no automatic firing)
  • Cooldowns must be respected (per session and per viewer)
  • The ⚡︎dB unlock cost must be clearly displayed before activation
  • Effects must auto-timeout or be reversible (no permanent overlays)
  • Never simulate ⚡︎dB activity (e.g., fake tip alerts) — all triggers must use the real NOIZ API

5. Audio & Visual Safety

For plugins that play audio or display visual effects:

  • Respect browser volume norms (no auto-blast sounds)
  • Warn users before flashing/strobing effects
  • Sudden loud or horror-trigger content must be opt-in only
  • Ambient or looping sounds must include a disable toggle
  • Content flagged for epilepsy risk, offensive language, or visual spam will be removed

6. Misuse & Exploit Handling

Plugins found to:

  • Hijack overlays
  • Trigger multiple effects per single ⚡︎dB unlock
  • Auto-activate on stream start without viewer input
  • Imitate core NOIZ UI or Partner/role badges
  • Trigger “fake events” to mislead viewers

…will be immediately delisted, and the developer may face:

  • Developer badge removal
  • Temporary or permanent Marketplace suspension
  • Account-level enforcement

7. Audit Process

NOIZ regularly audits active plugins for:

  • Security violations
  • Unexpected or undisclosed behavior
  • High or abnormal resource use
  • Viewer complaints or abuse reports

Developers may be asked to:

  • Patch their code
  • Disable specific features
  • Submit activity logs for review
  • Accept temporary unlisting during moderation

8. FAQ

Can I use third-party audio libraries? → Yes, if fully licensed and free of tracking components.

Can I store user settings between sessions? → Only through NOIZ-approved config APIs.

Can my plugin connect to outside websites? → Only with prior approval (e.g., for dynamic assets).

Can I monetize a plugin with crypto? → No — all monetization must use ⚡︎dB inside NOIZ.

What happens if I break these rules? → Plugin removal, Developer badge loss, and/or account action.
