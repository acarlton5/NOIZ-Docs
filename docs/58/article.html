2. Default Behavior (Auto-Routing)

By default, NOIZ uses smart automatic routing, which:

  • Picks the nearest verified relay with best health
  • Adapts mid-stream if a relay underperforms
  • Prioritizes Pathweaver or trusted relays in your location
  • Requires no setup ‚Äî it just works

3. Manual Relay Selection (Advanced Users)

Advanced users can override auto-routing by:

  • Navigating to NOIZ Studio ‚Üí Settings ‚Üí Relay
  • Choosing a node from the dropdown
  • Or pasting a custom ingest URL (if using OBS or other tools)

Manual routing is useful if:

  • You host a Pathweaver and want to lock to your own node
  • You're running a LAN stream, private session, or debug test
  • You need strict control for performance monitoring

Once locked manually, your stream stays on that node unless you restart or change the setting.

4. Using Your Own Relay Node

If you're hosting a Pathweaver relay, you can:

  • Select your node by its verified relay name
  • Lock it to serve your stream directly
  • View node diagnostics inside NOIZ Studio:
    • Ping
    • Bandwidth load
    • Node health status

Benefits include:

  • Regional priority for your relay
  • Direct connection to your stream viewers
  • Enhanced routing visibility and node preference

5. Fallback & Failover Behavior

If a relay node goes offline mid-stream:

  • NOIZ auto-reroutes you to the next best option
  • Viewers experience a brief buffer skip, not a stream drop
  • A notification appears in NOIZ Studio if rerouting occurs

Pathweavers can also:

  • Enable relay health alerts
  • Lock fallback options to regional clusters only

No action is needed unless you want stricter control.
