All NOIZ scene and overlay packs must meet platform compatibility, performance, and transparency standards. This policy protects streamers, promotes accessibility, and prevents malicious or unusable content from reaching the Marketplace.

1. Why These Guidelines Exist

All Marketplace scene layout submissions must:

  • Work reliably with NOIZ Studio and OBS 28+
  • Follow visual and accessibility best practices
  • Avoid performance-heavy or risky behavior
  • Disclose any plugin or third-party dependency

Applies to:

  • Scene templates & layout packs
  • Stingers and transitions
  • Starting/BRB/Ending screens
  • Plugin-integrated overlays

2. Basic Compatibility Requirements

All Marketplace scene assets must:

  • Be compatible with NOIZ Studio and standard OBS 28+
  • Use common base resolutions: 720p, 1080p, or 1440p
  • Stay within 16:9 safe zones
  • Include a README with clear setup steps

Submissions will be rejected if they:

  • Force a locked resolution (e.g. 1366x768-only)
  • Are designed only for ultra-wide or dual monitor layouts
  • Require heavy shaders or filters without fallback options

3. Required Folder Structure

Use the following directory format:

/YourOverlayPack/
├── README.md
├── preview.png
├── LICENSE.txt (optional)
├── overlays/
│   ├── cam_frame.png
│   ├── alerts/
│   ├── panels/
│   └── transitions/
├── scene_config.json or .overlay (if applicable)

README.md must include:

  • Setup instructions
  • Supported stream resolutions
  • Plugin compatibility (if applicable)
  • Artist or licensing notes

4. Performance & Accessibility Standards

Overlays must:

  • Use PNG, WebM, or SVG (compressed and GPU-safe)
  • Avoid autoplay audio or flashing effects unless toggleable
  • Be colorblind-aware (avoid red/green conflicts)
  • Stay under 10% average GPU usage on:
    • NVIDIA GTX 1060 or newer
    • Ryzen 5 3600 or similar
    • 8GB RAM minimum

Assets will be tested on NOIZ Studio’s baseline spec before approval.

5. Plugin Integration Guidelines

If your overlay includes plugin-driven or interactive components:

You must:

  • Use the NOIZ Plugin SDK v1+
  • Declare all external services, if any
  • Provide a plugin manifest + install guide
  • Include a clear permissions declaration

You must NOT:

  • Require third-party logins unless documented
  • Track users or store data without consent
  • Include hidden scripts, trackers, or unauthorized integrations

6. Design Best Practices

Highly recommended for creators:

  • Use modular layers (cam, chat, alerts = separate)
  • Provide light/dark theme variants
  • Leave margin for mobile-friendly scaling
  • Include editable source files (PSD, AE, etc.) — optional but appreciated
  • Ensure readable typography and text contrast

7. Prohibited Elements

Submissions will be rejected if they:

  • Include unlicensed game logos, streamer emotes, or third-party IP
  • Contain Twitch, YouTube, Kick, or other platform branding
  • Reference gambling, crypto, or adult service sponsors
  • Obscure the NOIZ watermark, plugin UI, or player branding

8. FAQ

Can I sell templates based on a game or fandom?

Only if you have commercial rights or the asset is generic (e.g., “retro sci-fi” is okay, “Zelda layout” is not).

Can I link to my commission or Patreon page?

Yes, in your Marketplace artist profile — not inside the overlay.

Can I include OBS filters or VST presets?

Yes — but warn users about OBS version compatibility.

Can I update my overlay after it's published?

Yes — version updates are supported and visible to all prior supporters.
