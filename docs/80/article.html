These guidelines outline visual, audio, and plugin-connected asset requirements for the NOIZ Marketplace to ensure quality, performance, and legal safety.

1. What These Guidelines Cover

This document outlines requirements for:

  • Static and animated overlay packs
  • Audio files for alerts, transitions, or stingers
  • Plugin-connected UI assets or effects
  • Combined visual + sound bundles

These rules ensure everything in the Marketplace:

  • Looks good on-stream
  • Performs well across devices
  • Is legally safe to use
  • Integrates cleanly into NOIZ Studio and compatible OBS setups

2. Visual Overlays & Scenes

Accepted formats:

  • PNG / WebP (static)
  • WebM / Lottie / SVG (animated)

Guidelines:

  • Max canvas: 1920x1080 unless explicitly scalable
  • Transparent where needed (e.g., camera frame windows)
  • Looping animations must be seamless
  • Total pack size should stay under 25MB per scene

Avoid:

  • Text baked into overlays (use dynamic plugin labels when possible)
  • Platform branding (NOIZ logos or Partner badges) unless officially sanctioned
  • Watermarks in final files (preview watermarks are fine)

3. Audio Alerts, Transitions & SFX

Accepted formats:

  • MP3 / WAV / OGG

Guidelines:

  • Under 10 seconds per file
  • No clipping, peaking, or distortion
  • Clearly categorized (e.g., alert tone, horror stinger, retro pop)
  • Volume normalized (target -14 LUFS)

Optional:

  • Only include original or license-free work
  • Label loop-friendly ambient sounds clearly

4. Plugin-Linked Assets

These are overlays or audio files designed to sync with NOIZ plugins, including:

  • Reaction flares
  • Quest notifications
  • Tip/sub event animations
  • Progress bars or token counters

Requirements:

  • Include setup or integration notes
  • Preview images should reflect in-plugin use
  • Assets must be responsive (scalable/adaptive for mobile)

Plugins must not:

  • Mislead users (e.g., fake alerts)
  • Use copyrighted game/show sounds
  • Break sandboxed layout areas

5. Bundling & Naming Conventions

For Marketplace clarity:

  • Group assets into themed unlock packs (e.g., “Cyberpunk Stream Kit”)
  • Include version/date tags if updated (e.g., v1.2, “Summer 2025”)
  • Use consistent filenames (alert_sub_loop.webm, brb_scene.png, sfx_ding.ogg)
  • Keep folder structures clean — avoid deeply nested directories

6. Tagging & Descriptions

Required for discovery:

  • Style tags: pixel, vaporwave, glitch, etc.
  • Function tags: starting scene, emote alert, SFX, etc.
  • Usage notes: “Works with Quest plugin,” “Optimized for mobile,” etc.
  • List of included files and formats

Avoid:

  • Misleading titles like “Twitch Animated Super Pack”
  • Using NOIZ brand names or Partner terms without authorization

7. Unlock & Access Notes

  • Marketplace items are unlocked by viewers using ⚡︎dB — they are digital-only platform assets, not direct purchases or physical goods
  • Artists receive ⚡︎dB allocations when their items are unlocked, based on their Artist Tier
  • Items cannot be transferred, exported for resale, or sublicensed without explicit licensing

8. FAQ

Can I use AI-generated art?
Only if fully edited and legally cleared for licensing
Can I submit free packs?
Yes — free listings still go through moderation
Can I upload extracted or modded game assets?
No — must be original or fully licensed
Do I need installation instructions?
Strongly encouraged, especially for plugin-reactive packs
