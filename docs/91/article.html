1. What Is the Developer Role on NOIZ

Developers create interactive and functional tools for the NOIZ ecosystem, such as:

  • Plugins (polls, games, alerts, minigames)
  • Overlay packs and UI extensions
  • Custom integrations for streamers
  • Tools that enhance viewer interaction or automate chat functions

There are two types of Developers:

RoleDescription
Community DevelopersExternal creators who build tools, plugins, or overlays — rewarded via Dev Quests or Marketplace ⚡︎dB unlocks
Staff DevelopersNOIZ employees who build core platform features or moderate plugin security

Only Community Developers carry the public Developer badge — identifying third-party contributors in chat and on profiles.

2. How Developers Are Rewarded

Developers can earn ⚡︎dB through:

  • Completing platform-assigned Dev Quests (feature bounties)
  • Listing interactive tools or plugins in the Marketplace
  • Collaborating with streamers for verified overlay integration
  • Participating in event-based builds (e.g., Hype Moments or Quest tie-ins)

⚡︎dB earned this way can be:

  • Cashed out if it’s charged and meets platform thresholds
  • Used on-platform to support others, unlock features, or fund personal projects

3. What Makes This Role Unique

Developers are not:

  • Staff or moderators
  • Automatically granted access to user data
  • Responsible for policy enforcement

Developers are:

  • Held to a higher technical and security standard
  • Trusted to follow Plugin Security & Access guidelines
  • Eligible for deeper API, SDK, and overlay tooling access
  • Invited to exclusive testing events and Dev Quests

4. Key Capabilities

As a Developer, you may:

  • Submit custom plugins to the Marketplace
  • Participate in seasonal or campaign-based Dev Quests
  • Access SDKs for real-time events, stream signals, or viewer inputs
  • Tag yourself as the builder of installed streamer assets
  • Earn the verified Developer badge (if approved)

5. Badge & Role Visibility

The Developer badge:

  • Appears in chat and profile when your work is published or active
  • Can be removed after extended inactivity or policy breach
  • Stacks with other roles (e.g., Pathweaver + Developer)
  • Represents trusted technical contribution, not employment or moderation authority

6. Who Can Apply

Anyone can apply to become a Developer if they:

  • Have a verified NOIZ account
  • Accept the SDK/Plugin Terms of Use
  • Complete at least one approved plugin or Dev Quest

You don’t need to be a streamer — developers can be viewers, artists, or independent coders with the required skills.

7. FAQ

Do I get paid in cash? → Only if you earn ⚡︎dB that is charged and meet the cashout threshold.

Do I need to open-source my code? → No, but open source is encouraged for transparency.

Can I build for specific streamers? → Yes, and you can list those builds in the Marketplace if approved.

Can I lose my Developer badge? → Yes, for security violations, plugin abuse, or prolonged inactivity.
