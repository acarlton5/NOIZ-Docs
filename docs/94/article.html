1. What These Terms Cover

These terms govern:

  • How you may use the NOIZ Plugin SDK
  • The acceptable behavior and structure of submitted plugins
  • What permissions and limitations apply
  • Enforcement consequences for violations

All developers must agree to these terms before accessing the SDK or submitting plugins to the Marketplace.

2. License & Usage Rights

  • The SDK is licensed for non-exclusive use on the NOIZ platform only
  • You may build plugins for:
    • Personal use
    • Streamer collaboration
    • Marketplace listing (via unlocks)
  • You may not distribute the SDK or its runtime components off-platform
  • Any tools built with the SDK must be submitted via the official Marketplace system before public use in streams
  • You retain full ownership of the code you write — but NOIZ retains platform-specific licensing to:
    • Run your plugin on streamer pages
    • Display plugin UI in overlays
    • Control distribution and security sandbox behavior

3. Prohibited Behaviors

Plugins may not:

  • Attempt to bypass NOIZ security sandboxing
  • Access personal viewer data, IP addresses, or browser fingerprints
  • Trigger ⚡︎dB actions without a viewer’s direct input
  • Play offensive content, loud audio, or flashing images without clear warnings
  • Fake stream events (e.g., simulate real tips or subs)
  • Contact unauthorized external APIs or embed off-platform analytics

Violations may result in plugin removal, loss of Developer badge, or account action.

4. Permitted Capabilities

Plugins may:

  • React to verified NOIZ stream events (via official hooks)
  • Request permissions to read from chat, overlay state, or viewer unlocks
  • Trigger visuals or sound if initiated by streamers/viewers
  • Offer configuration UIs for appearance or timing
  • Respect cooldowns, unlock costs, and trigger limits

All features must be declared in the plugin’s manifest file — undisclosed behavior will result in rejection.

5. ⚡︎dB & Monetization Restrictions

  • Plugins may be listed for ⚡︎dB unlocks, not for direct monetary sale
  • ⚡︎dB is a digital support token, not cash
  • Plugin unlocks are non-refundable
  • You may not charge viewers externally for plugin access
  • You may not accept crypto or fiat payments for plugin usage
  • All rewards and unlocks must occur within NOIZ, using approved systems

6. Data Handling & Privacy

Plugins must follow NOIZ’s platform-wide data policy:

  • No persistent storage beyond the active session
  • No tracking or fingerprinting
  • No attempts to infer personal identity from behavior
  • No reading of private chat messages, wallet details, or full inventory
  • Any plugin requiring access to viewer unlocks or input must prompt opt-in consent first

7. Updates & Enforcement

NOIZ reserves the right to:

  • Modify SDK structure, hooks, or runtime at any time
  • Disable or sandbox plugins that cause performance issues or policy violations
  • Enforce cooldowns, mute repeated triggers, or throttle heavy resource use

We may also flag developers for:

  • Submitting misleading or malicious plugins
  • Skipping required SDK components
  • Attempting to circumvent Marketplace moderation

8. FAQ

Can I fork the SDK? → Yes — for internal use only; you may not distribute modified versions.

Can I publish NOIZ plugins on other platforms? → No — the SDK is NOIZ-only.

Can I monetize plugins outside NOIZ? → No — monetization must occur within NOIZ using ⚡︎dB.

Can I bundle plugins with overlays or audio packs? → Yes — but all assets must be reviewed together.

What if someone copies my plugin? → Report it through the Marketplace — NOIZ will investigate and compare listings.
