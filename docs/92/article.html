1. What Is the NOIZ Plugin SDK

The NOIZ Plugin SDK allows Developers to build custom interactive features that streamers can add to their broadcasts, including:

  • Real-time overlays (polls, chat reactions, quest progress bars)
  • Viewer-triggered effects (⚡︎dB-based events, emote bursts, animations)
  • Token-based unlockables tied to stream activity
  • Custom HUD elements or gameplay integrations

All plugins run sandboxed in-browser or via NOIZ Studio and must follow platform-safe and security-approved practices.

2. Plugin Types

Plugin TypeExample Use Case
Overlay PluginsAnimated banners, follower counters, polls
Interaction Plugins⚡︎dB-triggered animations, emoji storms
Quest PluginsShow quest progress or viewer milestones
Audio PluginsPlay alert tones, ambient effects
Utility PluginsToken counters, timers, vote tallies

Plugins may run passively (always visible) or reactively (triggered by chat input, ⚡︎dB events, or platform signals).

3. SDK Access

To access the SDK, you must:

  1. Be a verified Developer
  2. Accept the Plugin SDK Terms
  3. Navigate to:
    • Developers → Plugin Builder → Get SDK

From there, you can:

  • Download or fork the boilerplate kit
  • Connect to:
    • Chat signals
    • Viewer ⚡︎dB triggers
    • Stream status (live, hype, featured, etc.)
    • Inventory unlock events

The SDK supports JavaScript (React, Vue, or Vanilla), WebSocket hooks, and the NOIZ Event API.

4. Plugin Lifecycle

Each plugin follows a structured process:

  1. Local Development — preview in dev mode
  2. Sandbox Testing — stream-safe environment without live chat posting
  3. Moderation Submission — for Marketplace listing or Quest integration
  4. Approval & Release — streamers can install the plugin
  5. Ongoing Updates — versioning and update notes visible to users

Plugins may be:

  • Free
  • Unlocked via ⚡︎dB
  • Included in platform event packs

5. Integration Features

The NOIZ Plugin SDK supports:

  • Reactive listeners — onTip, onSub, onQuestComplete, onKeyword
  • Event triggers — triggerOverlay, sendReaction, setCooldown
  • Chat-safe callbacks — chatEvent, viewerId, hasUnlock
  • UI placement options — dock, banner, fullscreen, pinned corner

Each plugin must clearly define:

  • Resource use (CPU/network)
  • Whether it uses audio or visual effects
  • Any required permissions

6. Security Model

Plugins are:

  • Sandboxed in secure iframes or OBS layers
  • Prohibited from collecting personal data or browser/device info
  • Blocked from external requests unless pre-approved
  • Audited both automatically and manually for security compliance

⚠️ Bypassing security layers will result in immediate removal and Developer badge revocation.

7. FAQ

Can I submit multiple plugins? → Yes, but each must go through approval individually.

Can I monetize my plugin? → Yes, via ⚡︎dB unlocks or Dev Quest rewards.

Can I update a plugin without review? → No — all changes require a moderation cycle.

Can streamers customize plugin visuals? → Yes — if you provide editable config options.

Is the SDK open source? → Core SDK tooling is public; runtime environment is proprietary.
