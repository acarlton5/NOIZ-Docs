1. What Is an Interactive Plugin?

An interactive plugin is any tool that responds to or triggers:

  • Viewer input (chat, clicks, reactions)
  • ⚡︎dB support actions (subs, cheers, quest completions)
  • Stream state (live/on break, featured, partner status)
  • Inventory unlocks or milestone events

Plugins enhance the stream experience and can appear:

  • As overlays
  • In chat
  • Inside viewer sidebars
  • Tied to Quest UIs or boost moments

2. Typical Plugin Use Cases

  • ⚡︎dB-triggered reactions (e.g. chat rain, explosion)
  • Stream polls or real-time voting widgets
  • Milestone or Quest completion trackers
  • Timer-based minigames or randomizers
  • Viewer name highlight effects
  • Collaborative goal bars (e.g. “raise 5k ⚡︎dB”)

If it engages the audience, you can build it.

3. Plugin Development Flow

Plan the interaction

  • What triggers it?
  • What does it display or do?
  • Does it need ⚡︎dB, chat, or viewer input?

Use the Plugin SDK Template

  • Fork or download the boilerplate
  • Customize manifest with trigger types, visuals, cooldowns
  • Hook into SDK events (e.g. onSub, onQuestComplete)

Preview and Test

  • Test in NOIZ’s local sandbox
  • Simulate events using mock viewer data
  • Optimize layout for both desktop and mobile viewers

Submit for Approval

  • Add metadata and preview images
  • Choose pricing (free or ⚡︎dB unlock)
  • Submit via Developer → Submit Plugin

4. Essential SDK Hooks

HookDescription
onTipViewer sends ⚡︎dB manually or via Sub
onQuestCompleteViewer completes a stream-linked Quest
onKeywordTriggers when a chat message matches input
triggerOverlay()Renders an overlay element (animation or HTML)
playSound()Triggers plugin-approved audio cue
setCooldown()Disables repeated spam (per viewer or global)

Every plugin must declare what hooks it uses in its manifest.

5. Plugin Design Best Practices

✅ Do:

  • Design for minimal input latency
  • Support chat-safe animations
  • Provide mobile-friendly scaling
  • Offer customization for streamers (color, text, placement)
  • Respect viewer input and never auto-trigger without action

❌ Don’t:

  • Auto-play sound or video on load
  • Simulate events (e.g. fake “you got a sub!”)
  • Manipulate chat logs or impersonate mods/staff
  • Store or transmit viewer data outside NOIZ

6. Testing Tools

Use the Plugin Sandbox tool to:

  • Emulate chat triggers
  • Simulate ⚡︎dB transactions
  • Test multiple viewer identities
  • Switch stream states (e.g. offline → live → featured)

If your plugin passes sandbox and visual review, you’re ready to submit.

7. Optimization & Performance

Keep assets:

  • Under 5MB for overlay bundles
  • Async-loaded where possible
  • Responsive to 720p–4K viewports
  • Compliant with security policy (sandboxed only)

⚠️ Memory-heavy or slow plugins will be rejected or throttled.

8. FAQ

Can I reuse Twitch or Kick plugin code? → Only if fully rewritten for NOIZ standards

Can I build audio-only tools? → Yes — audio plugins are supported, but must include volume controls

Can I partner with artists? → Absolutely — combine visuals + interactivity for maximum value

Can I test with real ⚡︎dB? → Test mode offers simulated ⚡︎dB — real usage requires streamer installation

Can my plugin store viewer data? → Plugins may temporarily cache viewer data only through NOIZ’s platform-native systems and never send or sync it externally. All data must remain session-scoped or stored in NOIZ-managed storage — no external communication, servers, or APIs are allowed.
